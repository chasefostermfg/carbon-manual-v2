Welcome to the **written Carbon manual** — the read-at-your-own-pace companion to
[Carbon Academy](https://learn.carbon.ms). Same curriculum, no video required. If you
have ten minutes between jobs, you can learn a whole module right here.

Carbon is an **open-source manufacturing system of record** — it combines an ERP
(the business: sales, purchasing, inventory, costing), an MES (the shop floor:
running jobs and recording work), and a QMS (quality: inspections, non-conformances
and traceability) into one connected system.

## Who this manual is for

This guide assumes **no prior ERP or manufacturing-software experience**. We define
every term the first time it appears (and collect them in the
[glossary](/manual/overview/glossary)), and every workflow is written as numbered
steps you can follow click-by-click. Whether you're an owner setting Carbon up, an
estimator building quotes, a buyer cutting purchase orders, or an operator on the
floor — there's a path here for you.

## The shape of Carbon

Most work in Carbon follows one thread. A customer asks for a price (a **quote**); you
win the work (a **sales order**); you make it (a **job** on the shop floor) and buy the
materials you need (a **purchase order**); you check it (**quality**); then you **ship**
it and **invoice** it. Everything else hangs off that spine.

<Mermaid
  chart={`flowchart LR
  Q[Quote] --> SO[Sales Order]
  SO --> J[Job / Production]
  J --> PO[Purchase Order for materials]
  PO --> J
  J --> QC[Quality check]
  QC --> SH[Shipment]
  SH --> INV[Sales Invoice]`}
/>

The [15-minute walkthrough](/manual/walkthrough) follows exactly this thread using one
real Acme part. After that, each section below goes deep on one stop along the way.

## How to read this manual

<Steps>
<Step>

</Step>
