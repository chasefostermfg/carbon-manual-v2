Tell Carbon a few facts about your business — who you are, where you keep stock, how you
measure things, how documents are numbered, and when you get paid. Set these up once, up
front. We'll use the real settings behind **Acme Inc.**, the example company in this manual.

## Where this lives in Carbon

Everything here lives in the **Settings** module. Open it from the **gear icon** at the
**bottom-left** of the navigation rail.

<AppShot
  src="/screenshots/manual/home-modules.png"
  alt="The Carbon home screen showing the module tiles and the navigation rail"
  caption="The Carbon home. Modules sit in the left rail — click the gear at the bottom-left to open Settings."
/>

Settings is split into sections: **Company**, **Locations**, **Units of Measure**,
**Sequences**, **Payment Terms**, and more. You'll set up five of them below.

<Callout type="tip">
  You need an [**administrator**](/manual/overview/glossary) account to change these settings.
  You can edit most of them later — except your **base currency**. Pick that one carefully.
</Callout>

## 1. Company profile

Your company profile is your business's ID card. Carbon stamps it onto every document you
send — quotes, order confirmations, purchase orders and invoices.

<Steps>
  <Step>
    Open **Settings → Company**.
  </Step>
  <Step>
    Enter your **name** and **address**. Acme is "Acme Inc." at 2400 Anvil Way, Springfield, OH.
  </Step>
  <Step>
    Upload your **logo** (PNG or SVG). It shows on PDFs and in the top-left of the app.
  </Step>
  <Step>
    Set your **base currency**. Acme uses **USD**. This is the currency your books are kept in.
  </Step>
  <Step>
    Click **Save**.
  </Step>
</Steps>

## 2. Locations

A **[location](/manual/overview/glossary)** is a physical site where you make things, hold
stock, or ship from. Carbon tracks inventory and work centers *per location*. Acme runs two.

<AppShot title="Settings · Locations" url="app.carbon.ms/settings/locations" caption="Acme's two operating locations." flush>
  <DataTable
    columns={[
      { key: "name", label: "Location", mono: true },
      { key: "role", label: "Role" },
      { key: "status", label: "Status", align: "right" },
    ]}
    rows={[
      { name: "Headquarters", role: "Main plant + warehouse — Springfield, OH", status: <StatusBadge tone="green">Active</StatusBadge> },
      { name: "Texas", role: "Satellite plant + warehouse", status: <StatusBadge tone="green">Active</StatusBadge> },
    ]}
  />
</AppShot>

- **Headquarters** is Acme's home base. It runs the Counterweight Hook thread you'll follow
  in this manual ([SO000069](/manual/selling/sales-orders),
  [J000097](/manual/manufacturing/jobs), [PO000057](/manual/buying/purchasing-basics)).
- **Texas** is a second site with its own burn table and laser table.

### Shelves and bins live inside a location

Inside a location you can say exactly *where* stock sits, using **storage units** (racks,
rows, levels, bins). Acme's A36 steel plate for the Hook job lives in bin **R02-B01-01**. You
don't need these on day one — one catch-all bin per location is enough to start.

<Callout type="tip">
  Add a location for every site that holds inventory or ships product. One building? One
  location is fine. Just don't lump two separate warehouses together, or your on-hand counts
  won't match reality. More in [Locations & stock](/manual/inventory/locations-and-stock).
</Callout>

## 3. Units of measure

A **unit of measure** (UoM) is *how you count a thing* — each, pounds, gallons, feet. Every
[item](/manual/overview/glossary) has one. The Counterweight Hook is counted in **EA** (Each);
the steel plate it's cut from is measured in **LB** (Pound). Acme runs 16 in total.

<AppShot title="Settings · Units of Measure" url="app.carbon.ms/settings/uom" caption="A selection of Acme's 16 units. EA and LB do most of the work." flush>

    ]}
  />

To add your own, open **Settings → Units of Measure** and give it a short **code** (the
abbreviation that shows on documents) and a **name**.

<Callout type="warn">
  Set an item's unit *before* you transact with it — it feeds pricing, inventory and
  purchasing. Buying in one unit and stocking in another (buy steel by the pound, receive it as
  plates) uses separate purchase and inventory units, not a change to the base unit.
</Callout>

## 4. Number sequences

Every document gets a short, readable ID — `Q000061`, `SO000069`, `J000097`, `PO000057`,
`AR000025`. You don't type these; Carbon builds each one from a **number sequence**: a prefix
plus a running counter.

<AppShot title="Settings · Sequences" url="app.carbon.ms/settings/sequences" caption="Each document type has a prefix and an auto-incrementing counter." flush>
  <DataTable
    columns={[
      { key: "doc", label: "Document" },
      { key: "prefix", label: "Prefix", mono: true },
      { key: "example", label: "Latest at Acme", mono: true },
      { key: "next", label: "Next", mono: true },
    ]}
    rows={[
      { doc: "Quote", prefix: "Q", example: "Q000063", next: "Q000064" },
      { doc: "Sales Order", prefix: "SO", example: "SO000071", next: "SO000072" },
      { doc: "Job", prefix: "J", example: "J000099", next: "J000100" },
      { doc: "Purchase Order", prefix: "PO", example: "PO000057", next: "PO000058" },
      { doc: "Sales Invoice", prefix: "AR", example: "AR000025", next: "AR000026" },
      { doc: "Purchase Invoice", prefix: "AP", example: "AP000027", next: "AP000028" },
    ]}
  />
</AppShot>

Each sequence has three parts you can tune:

- **Prefix** — the letters in front. **AR** is *accounts receivable* (money owed to you), **AP**
  is *accounts payable* (money you owe).
- **Padding** — how many digits, zero-filled. Acme pads to six, so `SO000069`, not `SO69`.
- **Next number** — the counter, which ticks up by one each time a document is created.

Acme's hero order flows through these sequences: quote **Q000061** becomes sales order
**SO000069**, spawns job **J000097**, triggers purchase order **PO000057**, and finishes
with invoice **AR000025**.

<Callout type="tip">
  Set prefixes and padding **before** you create your first document. The counter only goes up
  — don't reset it backwards, or you risk duplicate IDs.
</Callout>

## 5. Payment terms

**Payment terms** say *when* an invoice is due — and sometimes offer a discount for paying
early. Attach a default to each customer and supplier. Carbon copies it onto their orders and
invoices and works out the due date. Acme keeps seven.

**Net 30** is Acme's default and the term on the hero thread — on Radiant Aerospace's order
**SO000069**, the Ryerson purchase order **PO000057**, and the invoice **AR000025**.

<Callout type="info">
  Set one term as the company default, optionally override it on a specific customer or
  supplier, and Carbon fills it in automatically on every new order and invoice. You can still
  change it on one document when a deal calls for it.
</Callout>
