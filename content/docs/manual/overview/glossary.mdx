This page is your dictionary. Carbon uses a lot of words that sound technical but are simple
once someone explains them plainly. That's what this page does: one short definition per term,
plus a real example from **Acme Inc.** — the demo company used throughout this manual.

You don't need to read it top to bottom. Skim it once, then look things up as you go. Most pages
link their jargon straight here.

## Where this lives in Carbon

The glossary isn't a Module — it's a reference for all of them. Carbon is built from a set of
**Modules**: Sales, Purchasing, Items, Production, Inventory, Quality, Accounting, and Shop Floor.
Each definition below tells you which Module a term belongs to so you know where to click.

<Callout type="info">
  New to Carbon? Start with [What is Carbon?](/manual/overview/what-is-carbon) for the big
  picture, then follow one real order end to end in
  [Carbon in 15 minutes](/manual/walkthrough). The example used below — the *Counterweight Hook*
  `CWH-A36-15` that Acme makes for Radiant Aerospace — is that same order.
</Callout>

## The three letters: ERP, MES, QMS

Carbon is "an ERP + MES + QMS." Each acronym covers one slice of running a shop. Carbon rolls
all three into one system that shares one set of data.

<DataTable
  columns={[
    { key: "term", label: "Term", mono: true },
    { key: "stands", label: "Stands for" },
    { key: "means", label: "What it does for you" },
  ]}
  rows={[
    { term: "ERP", stands: "Enterprise Resource Planning", means: "The business side: customers, suppliers, sales orders, purchase orders, invoices, inventory and costs." },
    { term: "MES", stands: "Manufacturing Execution System", means: "The shop-floor side: what operators do — clock onto a job, record good and scrapped pieces, follow work instructions." },
    { term: "QMS", stands: "Quality Management System", means: "The quality side: inspections, non-conformances, corrective actions, gauge calibration and full traceability." },
  ]}
/>

Because they share one system, finishing a shop-floor step (MES) updates the job cost (ERP) and
records which material went in (QMS) — all at once, with no re-typing.

## How the big terms connect

If a few terms still feel abstract, here's the chain they form in real life. Each box is a record
in Carbon, and each one links to the next automatically — the same thread the
[walkthrough](/manual/walkthrough) follows.

<Mermaid
  chart={`flowchart LR
  Item[Item CWH-A36-15] --> Method[Method BOM plus Routing]
  Quote[Quote Q000061] --> SO[Sales Order SO000069]
  SO --> Job[Job J000097]
  Method --> Job
  Job --> PO[Purchase Order PO000057]
  Job --> Ship[Shipment]
  Ship --> Invoice[Sales Invoice AR000025]`}
/>

<Callout type="tip">
  A quick way to remember the item types: **Part** and **Material** become the product (one is
  made/sold, one is consumed into it), while **Tool**, **Fixture** and **Consumable** are what
  you use to do the work. A **Service** is work itself rather than a thing.
</Callout>
